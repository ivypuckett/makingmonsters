## Commands

Use [Taskfile](https://taskfile.dev/) from the repo root, or npm scripts from `client/`.

Tasks are defined in per-directory Taskfiles (`client/`, `server/`, `iac/`) and included into the root Taskfile under their respective namespaces. Dependent tasks (e.g. `install` before `dev`) run automatically ‚Äî you only need to invoke the top-level task.

**Frontend**

| Task command | Description |
|---|---|
| `task client:install` | Install frontend dependencies |
| `task client:dev` | Start dev server with HMR (auto-runs `install`) |
| `task client:build` | Production build ‚Üí `client/dist/` (auto-runs `install`) |
| `task client:preview` | Preview production build (auto-runs `build`) |

**Server**

| Task command | Description |
|---|---|
| `task server:init` | Download Go dependencies (`go mod tidy`) ‚Äî run once after cloning |
| `task server:build` | Compile the PocketBase binary to `server/pocketbase` (auto-runs `init`) |
| `task server:dev` | Run PocketBase locally on `:8090` |

**Infrastructure**

| Task command | Description |
|---|---|
| `task iac:init` | Initialize OpenTofu providers ‚Äî run once |
| `task iac:plan` | Preview infrastructure changes |
| `task iac:apply` | Apply infrastructure changes |
| `task deploy` | Build frontend then deploy full stack to fly.io (auto-runs `client:build`) |

Run `task` with no arguments to list all available tasks.

## Code Conventions

### File naming
- Svelte components: `PascalCase.svelte` (e.g., `Counter.svelte`)
- JavaScript files: `camelCase.js` (e.g., `main.js`)
- CSS files: `kebab-case.css` (e.g., `app.css`)

### Svelte
- Use **Svelte 5 runes** for reactivity (`$state`, `$derived`, `$effect`)
- Do not use Svelte 4 `$:` reactive declarations or `writable`/`readable` stores
- Keep components in `src/lib/` for reusability; page-level components go in `src/`

### Styling
- Use **Tailwind utility classes** for layout and spacing
- Use **DaisyUI component classes** for pre-styled UI elements (buttons, cards, etc.)
- Global styles (e.g., `@import "tailwindcss"`, plugin declarations) go in `src/app.css`
- Avoid writing custom CSS unless Tailwind/DaisyUI cannot cover the case

### JavaScript
- ES modules throughout (`"type": "module"` in package.json)
- `verbatimModuleSyntax` is enabled ‚Äî use `import type` for type-only imports
- `checkJs` is enabled ‚Äî keep code type-safe without requiring `.ts` extensions

## Development Workflow

### Frontend only
1. `task client:dev` ‚Äî installs dependencies automatically, then starts dev server with HMR
2. Make changes ‚Äî Svelte component state is preserved across HMR updates unless the `<script>` block changes
3. `task client:build` ‚Äî production build to `client/dist/`

### Full stack (frontend + PocketBase)
1. `task client:build` ‚Äî installs frontend dependencies then builds Svelte into `client/dist/`
2. `task server:dev` ‚Äî fetches Go dependencies if needed, then runs PocketBase on `http://localhost:8090`; admin UI at `http://localhost:8090/_/`

### Server conventions
- Custom routes and hooks go in `server/main.go` (or additional `.go` files in `server/`)
- PocketBase stores its data in `pb_data/` (excluded from git via `.gitignore`)
- PocketBase serves the built Svelte frontend from `pb_public/` ‚Äî always run `task client:build` first
